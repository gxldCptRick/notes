Maven Web App Archetype

So we learned about a new archetype that we can use to create a Web App and have it just work with maven. The thing is that it doesn't include some pretty important stuff and so we would have to do some setup so that we can quickly get up and running with it.

Create a Maven Project to work on whatever

Click on the next button so that you can select a maven archetype

Then search for maven-archetype-webapp and select the archetype

Fill out the info for your maven project and blam you then have to actually fix the project it created

Right click on the project you just created and then go to properties

Go to the project facets and then go into the runtime tabs.

Click on the runtime that we want and it should fix the first problem we have.

Now we have to create the folder for our sources so we will create a java folder inside the main folder.

After all the steps you will have finally created a full maven powered web project.

JPA Entity Manager

After that we got started on setting up the JPA Entity Manager stuff. The thing is that Mr. Cox couldn't figure out why the Entity Manager could not find the configuration file it needed to make it work. He also published an announcement with the file info if you want to check it out. Because of that we actually didn't get it all the things setup today and he was planning on getting it done by the time we would have a next assignment. So theoretically it would work if you add the persistance.xml inside of the META-INF folder.

Link To Persistance Link to Announcement 
          
<persistence xmlns="http://xmlns.jcp.org/xml/ns/persistence" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/persistence
http://xmlns.jcp.org/xml/ns/persistence/persistence_2_1.xsd" version="2.1">
    <persistence-unit name="ps">
        <description>
          Hibernate Entity Manager Example
        </description>
        <!-- using entity manager with hibernate  -->
        <!-- hibernate is implementation we will be using of the JPA -->
        <provider>
          org.hibernate.jpa.HibernatePersistenceProvider
        </provider>
        <properties>
            <!-- tells hibernate how to connect -->
            <!-- this is where you driver is -->
            <property name="javax.persistence.jdbc.driver" value="org.mariadb.jdbc.Driver" />
            <!-- this is the url to connect with. -->
            <!-- mariadb connects by default on 3306 but we can specify a different port -->
            <property name="javax.persistence.jdbc.url" value="jdbc:mariadb://localhost:3307/CSC280_people" />
            <!-- setting the username -->
            <property name="javax.persistence.jdbc.user" value="CSC280_people" />
            <!-- setting the password -->
            <property name="javax.persistence.jdbc.password" value="CSC280_people" />
            <!-- this is an optional property it is useful debugging -->
            <!-- it allows you to see the sql statements that got generated -->
            <property name="hibernate.show_sql" value="true" />
        </properties>
    </persistence-unit>   
</persistence>
